[SLIDE 9: PractaThon Checkpoint - Impact Quantification]

**NARRATION:**

"Your checkpoint exercise before M13.4: Quantify the business impact of your system.

**Learn (5-10 min):**
- Review your M13.1-M13.3 work and identify measurable outcomes
- Study 3 senior engineer resumes/portfolios to see how they present impact
- Research target companies and roles you'll apply to

**Build (10-15 min):**
- Create impact metrics spreadsheet:
  * Scale achieved: X req/hour, Y tenants, Z uptime
  * Cost savings: Reduced infrastructure cost from $A to $B
  * Performance: P95 latency <Xms, error rate <Y%
  * Business: System could serve $X ARR at current scale
- Draft 3-sentence project description focusing on outcomes, not features
- Screenshot key metrics: Grafana dashboards, load test results, compliance docs count

**Apply (10-15 min):**
- Calculate specific numbers:
  * Your system vs. manually searching 10,000 docs: X hours saved per query
  * Cost to run at 10K req/hour: $Y/month
  * Tenant onboarding time: X minutes (automated vs. manual)
- Test your impact statement with 2-3 engineer friends: "Does this impress you?"

**Ship (5 min):**
- Document all metrics in `docs/impact-metrics.md`
- Include screenshots in `docs/screenshots/` folder
- Commit to GitHub
